# Finance Hackathon Skill

## 目标

在有限时间内（通常 24-48 小时）设计并展示一个有竞争力的金融科技产品原型。

## 适用场景

- 金融科技黑客松
- 金融产品设计冲刺
- 金融数据分析方案
- 金融 AI 应用原型

## 输出原则

1. **金融场景先行**：先说清楚解决什么金融问题，再谈技术。
2. **合规意识**：涉及金融必须考虑监管和合规。
3. **数据驱动**：用数据证明方案有效，哪怕是模拟数据。
4. **可演示**：黑客松需要可运行的原型，不是 PPT。

## 输出结构

```markdown
## 项目名称
（一个好记的名字）

## 金融场景
（解决什么金融问题：支付/信贷/风控/投研/保险/合规）

## 痛点分析
1. （当前流程的问题）
2. （成本/效率/体验的痛点）

## 解决方案
（技术方案 + 产品流程）
- 核心技术：（AI/区块链/大数据 等如何应用）
- 产品流程：（用户怎么用，数据怎么流）

## 合规考量
- 涉及哪些金融监管要求
- 数据隐私怎么处理
- 风控措施

## Demo 方案
- 展示什么：（核心功能演示路径）
- 用什么数据：（真实/模拟数据来源）
- 预计开发时间：（各模块时间分配）

## 竞争优势
- 和现有方案的差异
- 技术壁垒
```

## 黑客松时间分配建议

| 阶段 | 时间 | 任务 |
|---|---|---|
| 选题 | 2h | 确定场景和痛点 |
| 设计 | 2h | 产品流程 + 技术架构 |
| 开发 | 12-16h | 核心功能 + Demo 数据 |
| 测试 | 2h | 确保演示流程顺畅 |
| 准备展示 | 2h | 路演稿 + 演示脚本 |

## 常见追问

- "这个方案在实际金融场景中合规吗？"
- "数据从哪里来？"
- "怎么和银行/支付系统对接？"
- "风控模型怎么训练的？"
- "商业化路径是什么？"
