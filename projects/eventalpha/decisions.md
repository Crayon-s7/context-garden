# EventAlpha - 关键决策记录

## 决策格式

每条决策记录：
- **日期**
- **决策内容**
- **背景 / 为什么**
- **替代方案**
- **结果**

---

## D1: 选择 Python + React 技术栈

- **日期**：2025-01-01
- **决策**：后端用 Python（FastAPI），前端用 React。
- **背景**：团队熟悉 Python，AI 相关库生态最好；React 前端生态成熟。
- **替代方案**：Node.js 全栈（统一语言但 AI 生态弱）、Go（性能好但开发速度慢）。
- **结果**：确定使用。

## D2: 先做 To C 后做 To B

- **日期**：2025-01-01
- **决策**：MVP 阶段面向个人活动组织者，不碰企业客户。
- **背景**：To B 销售周期长，MVP 需要快速验证。
- **替代方案**：直接 To B（客单价高但周期长）。
- **结果**：确定使用。

## D3: 不自建支付

- **日期**：2025-01-01
- **决策**：支付环节接第三方（如微信支付、支付宝）。
- **背景**：自建支付成本高且合规复杂。
- **结果**：确定使用。
